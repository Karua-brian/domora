# Engineering Handbook

## Rule #1
Business before technology.

## Rule #2
The Domain is independent of frameworks.

## Rule #3
Every file must justify its existence.

## Rule #4
Documentation precedes implementation.

## Rule #5
A repository must explain itself.

## Rule #6
Delete examples, not infrastructure.

## Rule #7
Every commit tells a story.

## Rule #8
The repository must always build.

## Rule #9
Optimize for discoverability, not visual order.

## Rule #10
Folders represent business capabilities, not technical categories.
